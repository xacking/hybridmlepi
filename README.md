# hybridmlepi ‚Äî Explainable Ensemble Deep Learning for Early Epidemic Prediction

Code and derived results for an explainable ensemble framework that couples a
multi-kernel **CNN** and a **(Bi)LSTM** feature extractor with a **Random Forest**
classifier, applied to two epidemic-surveillance tasks:

- **Social-media track** ‚Äî sentiment-augmented detection of symptom-bearing
  ("warning") tweets during the 2022 Ugandan Ebola outbreak.
- **Clinical track** ‚Äî pre-PCR COVID-19 triage from routine complete-blood-count
  (CBC) features, with **SHAP** (TreeExplainer) and permutation-importance
  explainability.

> **Reproducibility note.** Every figure and metric in the manuscript is
> regenerated by the notebooks below. Notebook outputs have been cleared from
> this repository (to comply with data-redistribution terms ‚Äî see
> [`data/README.md`](data/README.md)); re-run the notebooks against the source
> data to reproduce them.

The model progression in each track is a controlled ablation:
`CNN‚ÜíRF` ‚Üí `+ sentiment/clinical features` ‚Üí `CNN+LSTM (+features) ‚Üí RF`.

## Data

Raw datasets are **not** redistributed here. See [`data/README.md`](data/README.md)
for how to obtain them:

- **Ebola tweets** ‚Äî third-party corpus (Mirugwe et al., 2024); under X/Twitter
  terms only tweet IDs + a hydration route may be shared, not raw tweet text.
- **COVID-19 CBC** ‚Äî publicly available on Kaggle (Hospital Israelita Albert
  Einstein, de-identified under LGPD); cite the source rather than re-hosting.

## Setup

```bash
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
python -m nltk.downloader stopwords punkt wordnet vader_lexicon
jupyter lab
```

Tested with Python 3.11, scikit-learn 1.7, SHAP 0.49.

## How to reproduce

1. Place the source data as described in [`data/README.md`](data/README.md).
2. Run the notebooks in ablation order within each track (v2.0/2.1 ‚Üí v2.3).
3. Generated metrics and figures will match the files in `results/`.
